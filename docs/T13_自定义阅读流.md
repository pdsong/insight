# T13 — 自定义阅读流

## 完成时间
2026-02-26

## 做了什么

### 1. `Feeds` 上下文 (Context)
- 利用 `CustomFeed` (name, rules) 实现了标准 CRUD 操作。
- 增加了核心的 `query_feed/2` 查询引擎，用于动态解析 JSON `rules` 为 Ecto 查询。
- **查询能力**：
  - `tags`: 包含至少一个列表内标签名的新闻（通过 `news_tags` 子查询实现与 SQLite 的兼容）。
  - `keywords`: 新闻的标题（或中文标题） LIKE 匹配至少一个给定的关键词列表。
  - **组合逻辑**：如果两者都提供，按 **AND** 处理（符合包含标签 AND 匹配关键词条件）。

### 2. `FeedLive.Index` 管理页面
- 新增页面并在 `/feeds` 提供独立路由。
- **创建/编辑功能**：设定名字，多选系统和个人现有的标签，并用逗号输入自定义关键词组合。
- 采用 DaisyUI 展示每个 Feed 的状态标签，提供一键跳转至该 feed 或者管理更新的修改按钮。

### 3. 与主页 (`NewsLive.Index`) 的功能集成
- 通过对 `?feed=<id>` URL 查询重定向到专门查询该 `id` 阅读流的子状态。
- 原有的热度/时间类导航栏扩展了自定义的 Tag，用户可以一键点选已定义好的自定义阅读流。
- 原本针对全局列表触发的分页和阅读等交互同样适用特定自定义流内。
