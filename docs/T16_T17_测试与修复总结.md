## 做了什么

### 1. 修复由于新功能引入的测试环境 Bug
- **修复 SQLite3 并发测试死锁问题 (Database busy)**：由于 T14 的隐式行为画像更新使用了 `Task.start`，其在并行的 ExUnit 环境及 SQL Sandbox 孤立事务机制下会诱发 SQLite 库由于并发导致的死锁。我们给 `Insight.Interactions` 增加了针对 `Mix.env() == :test` 的判断，在测试环境中拦截其退回同步执行 (`calculate_all_user_interest_profiles_async`)，解决数据库占用堵塞。
- **修复参数赋值上下文不匹配 (KeyError)**：修复了 `UserLive.Radar` 和 `UserLive.DailySummary` 中误将 `socket.assigns.current_scope.user_id` 当作现有字段访问的 Bug，纠正为了 `socket.assigns.current_scope.user.id`。

### 2. 补充 T16 & T17 单元与集成测试
为了保证这部分重度数据渲染与 UI 交互模块的健壮性，我们补充了一系列针对 T16 / T17 功能维度的测试用例，并全部验证通过：
- `test/insight/interactions/stats_test.exs`: 测试成就头衔称号算法和阅读标签分布计算。
- `test/insight/news/daily_summary_test.exs`: 测试 `Insight.News.DailySummary` 的存取行为及历史按序提取。
- `test/insight_web/live/user_live/daily_summary_test.exs`: 测试 `/daily-summaries` 页面的侧边栏历史切换、空状态，并处理 LiveView 断言时由于 `~p` 不能接受 `%Date{}` Struct 抛出的 `ArgumentError`。
- `test/insight_web/live/user_live/radar_test.exs`: 测试 `/radar` 技术雷达的成就面板载入和默认空值 UI 展现。

通过执行 `mix test`，系统累计测试达到 184 个项点，零报错 (184 tests, 0 failures)。所有新添加的 T16 与 T17 指标已稳定汇入主分支的主测试套件。
