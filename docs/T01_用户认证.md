# T01 — 用户注册/登录/邮箱重置密码

## 完成时间
2026-02-25

## 做了什么

### 1. 用户认证系统搭建
利用 Phoenix 1.8 内置的 `mix phx.gen.auth Accounts User users --live` 生成完整的认证脚手架，包括：

- **注册**：用户输入邮箱创建账号，系统自动发送确认邮件
- **登录**：支持两种方式 — 邮箱魔法链接（无密码）和传统的邮箱 + 密码
- **登出**：清除 session 和 remember_me cookie
- **密码重置**：在设置页面修改密码
- **邮箱修改**：修改邮箱后发送确认链接到新邮箱
- **Sudo 模式**：访问敏感页面（如设置）需要近期认证过

### 2. 中文化 & 品牌定制
将所有认证页面的界面文字翻译为中文：
- `login.ex` — 登录页面
- `registration.ex` — 注册页面
- `confirmation.ex` — 邮箱确认页面
- `settings.ex` — 账号设置页面
- `root.html.heex` — 顶部导航栏（注册/登录/设置/退出）
- `layouts.ex` — 应用导航栏品牌名"Insight 智能阅读"

### 3. 代码注释
为每个 LiveView 模块添加了 `@moduledoc`（模块功能说明）和关键代码行的中文注释。

### 4. 测试更新
将 4 个 LiveView 测试文件中的英文断言更新为中文，确保与 UI 文字一致。

## 碰到了什么困难

### 问题 1：数据库迁移冲突
`mix ecto.migrate` 报错 `table "users" already exists`，原因是数据库中已有旧表但 migration 记录不存在。
**解决方案**：使用 `mix ecto.reset` 先删库再重建，dev 和 test 环境都需要重置。

### 问题 2：测试大面积失败（23/111）
修改 UI 文字为中文后，测试中所有 `assert html =~ "English text"` 全部失败。
**解决方案**：逐个更新测试断言，将英文字符串替换为对应中文。注意区分两类断言：
- **模板渲染文字**（如 "Log in" → "登录"）— 需要更新
- **服务端 Flash 消息**（如 "Invalid email or password"）— 保持英文不变，因为这些来自 `UserSessionController`，不在 LiveView 模板中

## 初学者知识点

### Phoenix.gen.auth 做了什么？
这个生成器自动创建了完整的认证系统：
- `Accounts` context — 用户 CRUD 和认证业务逻辑
- `User` schema — 用户数据模型（email, hashed_password）
- `UserToken` schema — 用于 session、魔法链接、邮箱确认的令牌
- `UserAuth` plug — 认证相关的 Plug（fetch_current_scope、require_authenticated 等）
- 4 个 LiveView 页面 — 注册、登录、确认、设置
- 完整的测试套件 — 111 个测试覆盖所有认证场景

### Sudo 模式是什么？
为了保护敏感操作（如修改密码/邮箱），Phoenix 1.8 引入了"sudo 模式"。用户必须在最近 10 分钟内通过认证（登录或重新验证），否则访问设置页面时会被要求重新登录。

### 魔法链接 vs 密码登录
Phoenix 1.8 的认证系统默认支持"无密码登录"（Magic Link）。用户只需输入邮箱，系统发送一个一次性链接到邮箱，点击即可登录。密码是可选的，用户可以在设置中启用。
