# T16 — 个人日报 & 周报

## 完成时间
2026-02-26

## 做了什么

### 1. DailySummary Schema & Migration
- 新增了 `Insight.News.DailySummary` 模型，记录字段：`user_id`，`date`，`content` (Markdown格式的大模型回复)，`status` (`pending`, `generating`, `completed`, `failed`)。
- 在 `Insight.News` 上下文中增加了 `upsert_daily_summary/1`、`list_daily_summaries/1` 和 `get_daily_summary/2`。

### 2. 调度器与拉取逻辑
- 创建了 `Insight.News.DailySummaryWorker` 的定时 `GenServer`。后台每小时做一次检查，若到达指定时间（比如北京时间上午 9 点之后）且该用户当日简报还未生成，则派发后台生成任务。
- 添加 `Insight.News.DailySummaryGenerator.generate_all_summaries/0` 和针对单个用户的生成器：
  - 加载用户 Top 3 兴趣画像。
  - 从 `NewsItem` 去取过去 24 小时内抓取的包含该兴趣关联 Tag 的至多 15 篇新闻。
  - 自动通过 `Insight.AI.DailySummarizer.generate_newsletter/2` 调优 Qwen 提示词生成一份 Markdown 排版的日报。

### 3. Web 端：用户个人日报查阅页面
- 在路由中加入了 `/daily-summaries` 并在顶导引入 `个人日报` 的入口链接。
- 新增了 `InsightWeb.UserLive.DailySummary` 页面：
  - 左侧边栏列出历史简报，带时间戳和状态指示 `(生成中/已完成/失败)`。
  - 右侧主视图解析从数据库传回的 Markdown 内容。引入了新依赖 `Earmark` 来将 AI 返回的美丽 Markdown 渲染成 HTML (`{raw Earmark.as_html!(@selected_summary.content)}`)，配以 `@tailwindcss/typography` 的 `prose` 渲染出优雅的文章质感。
