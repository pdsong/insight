# T15 — "Serendipity" 破圈推荐

## 完成时间
2026-02-26

## 做了什么

### 1. 破圈新闻选择逻辑
- 在 `Insight.News: fetch_serendipity_news/4` 中实现。
- 策略：查询最近的 200 条新闻，剔除属于用户当前核心兴趣（Top 3 Profiles）关联标签的新闻，以及剔除当前正在被展出的新闻（防止卡片重复渲染冲突）。
- 从剩余内容中随机 `Enum.shuffle` 取出 2 篇。

### 2. 破圈推荐语生成 (AI)
- 在 `Insight.AI.Recommender` 中新增了 `generate_serendipity_reason/1`。
- 提示词特定：要求生成语气像朋友推荐，强调“跨界探索”、“新知”、“拓宽视野”或“偶然的惊喜”（例如：“跳出日常，这篇关于...的文章或许能给你新的灵感”）。
100 字内的高转化文案。

### 3. 信息流融合展示
- 在首页默认视图（未带任何特殊筛选和搜索参数的第 1 页）中调用此逻辑。
- 拿到 2 篇 serendipity 文章，并打上虚拟标记 `is_serendipity: true`，固定插入在原信息流的第 3 个和第 7 个位置。
- 利用并行的 `Task.async` 向 AI 后台获取破圈推荐语。
- **自定义卡片样式**：使用 `bg-gradient-to-r from-purple-100/50 to-pink-100/50` 背景，卡片右上角悬浮显眼的 `破圈推荐` 渐变小 Badge，推荐语前带有 ✨ (hero-sparkles) 让视觉呈现更灵动。

## 测试结果
- 修复了因为重复数据导致 LiveView Test 断言匹配出两个元素的 Bug，将 `exclude_news_ids` 参数传入底层屏蔽冲突 ID。
- 测试正常，170 tests, 0 failures 保持稳定。
